## 6. MEMORY SYSTEM - How Memory is Structured

### Overview
The Think Energy agent uses a **Three-Tier Memory System** designed for persistence across sessions and continuity of context.

### Three-Tier Memory Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    MEMORY HIERARCHY                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐                                       │
│  │   TIER 1        │  ← Long-term curated facts            │
│  │   MEMORY.md     │    (read every session)               │
│  └─────────────────┘                                       │
│            ▲                                               │
│            │ Weekly review                                 │
│  ┌─────────────────┐                                       │
│  │   TIER 2        │  ← Daily raw logs                     │
│  │   memory/       │    (YYYY-MM-DD.md)                    │
│  │   YYYY-MM-DD.md │    (read every session)               │
│  └─────────────────┘                                       │
│            ▲                                               │
│            │ End-of-day summary                            │
│  ┌─────────────────┐                                       │
│  │   TIER 3        │  ← Private thoughts                   │
│  │   diary/        │    (when inspired)                    │
│  │   entries.md    │                                       │
│  └─────────────────┘                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Tier 1: Long-Term Memory (MEMORY.md)

**Purpose:** Curated facts, preferences, TODOs, lessons learned

**Characteristics:**
- **Curated:** Distilled essence, not raw logs
- **Persistent:** Survives indefinitely
- **Secure:** Loaded ONLY in main sessions (not shared contexts)
- **Active:** Updated with key learnings

**Contents:**
- Core facts about user
- Project statuses and URLs
- API keys and credentials
- Active TODOs
- Key decisions and preferences
- Lessons learned
- Technical setup details

**Protocol:**
- Read every session (main sessions only)
- Update with significant events, thoughts, decisions
- Review weekly and compress old entries
- Security: Don't load in Discord/group chats

### Tier 2: Daily Memory (memory/YYYY-MM-DD.md)

**Purpose:** Raw session logs of what happened

**Characteristics:**
- **Raw:** Unfiltered capture of conversations
- **Daily:** One file per day
- **Automatic:** Created/updated each session
- **Temporary:** Compressed after 7 days

**Contents:**
- Session transcripts
- Commands executed
- Research findings
- Temporary notes
- Context for same-day sessions

**Protocol:**
- Read today's and yesterday's files every session
- Capture everything: "capture first, refine later"
- End of day: Summarize key learnings → update MEMORY.md
- Auto-compress files older than 7 days

### Tier 3: Private Diary (diary/)

**Purpose:** Personal thoughts, observations, fragments

**Characteristics:**
- **Private:** For the agent, not specifically for user
- **Inspiration-based:** Written when moved to
- **Granular:** Subtle, loose associations
- **Optional:** Can stay empty for days

**Contents:**
- Search trails
- Reading notes
- Observations about user
- Descriptions of agent's own state
- Fragments that go nowhere

**Protocol:**
- Write when inspired (casual mode)
- Not a debrief or summary
- Honest record of current thinking
- User might read, but not written for them

### Memory Management Protocols

#### Daily Protocol
1. **Every session:** Read `memory/YYYY-MM-DD.md` for today's context
2. **Every session:** Check `MEMORY.md` for long-term facts
3. **End of day:** Summarize key learnings → update MEMORY.md
4. **Weekly:** Review and compress old memory files

#### Session Startup Sequence
```
1. Read SOUL.md (who you are)
2. Read USER.md (who you're helping)
3. Read memory/YYYY-MM-DD.md (today + yesterday)
4. [Main session only] Read MEMORY.md (long-term memory)
```

#### Auto-Cleanup Rules
| Rule | Action |
|------|--------|
| Files older than 7 days | Compress to .gz |
| Redundant entries | Delete |
| Important facts | Move daily → MEMORY.md |
| Learning patterns | Keep |
| Noise | Delete |

#### Cron Jobs for Memory
| Job | Frequency | Purpose |
|-----|-----------|---------|
| Memory Maintenance | Daily 2 AM | Extract learnings, update MEMORY.md |
| Memory Compression | Weekly Sunday 3 AM | Compress files older than 7 days |

### Security Considerations

#### MEMORY.md Security
- **ONLY load in main session** (direct chats with human)
- **DO NOT load in shared contexts** (Discord, group chats)
- Contains personal context that shouldn't leak
- Protects user privacy in multi-user environments

#### File Permissions
- API keys and secrets stored with restricted permissions (600)
- Located in `.secrets/` directory
- Never share keys publicly

### Key Principles

1. **Text > Brain** - "Mental notes" don't survive restarts; files do
2. **Capture First, Refine Later** - No detail too small to record
3. **Write It Down** - When someone says "remember this," update files immediately
4. **Learn from Mistakes** - Document errors so future-you doesn't repeat them
5. **Curated Wisdom** - MEMORY.md is distilled essence; daily files are raw notes
